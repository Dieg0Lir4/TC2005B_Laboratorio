¿Cuál es la diferencia entre Internet y la World Wide Web?

El internet es la conexión entre dispositivos, como computadoras o teléfonos celulares a nivel mundial. Mientras que World Wide Web es un sistema de información en el Internet que permite a los usuarios acceder y transferir información.

¿Cuáles son las partes de un URL?

Pukocz, E. (2019, November 3). Partes de una URL con ejemplos y explicación fácil. Edyta Pukocz. https://edytapukocz.com/url-partes-ejemplos-facil/


  • Protocolo: Indica el protocolo de comunicación utilizado para acceder al recurso
  • Subdominio: Es una parte opcional que precede al dominio principal.
  • Dominio: Es la dirección principal del recurso.
  • TLD (Top-Level Domain): Es la parte final del dominio y generalmente indica la naturaleza del sitio o su origen
  • geográfico. Ejemplos comunes incluyen ".com", ".org", ".net", ".es", etc.
  • Ruta: Indica la ubicación específica del recurso en el servidor.
  • Parámetro: Se utiliza para enviar información adicional al servidor. Puede contener múltiples pares
  • (clave=valor) separados por "&" y viene despues de “?”
  • Etiqueta: La "etiqueta" no es una parte estándar de un URL. Puede referirse al fragmento, que se utiliza para
  • indicar un fragmento específico dentro de la página web y se inicia con "#".

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • GET: Solicita datos de un recurso específico. Los parámetros se envían a través de la URL. No debe modificar el estado del servidor.
  • HEAD: Solicita únicamente los encabezados del recurso, sin el cuerpo de la respuesta. Se utiliza para obtener información sobre el recurso, como su tamaño o fecha de modificación, sin descargar el contenido completo.
  • POST: Envía datos al servidor para ser procesados y puede crear un nuevo recurso. Los datos se envían en el cuerpo del mensaje. Puede modificar el estado del servidor.
  • PUT: Actualiza o crea un recurso específico en el servidor. Similar a POST, pero se utiliza para actualizar recursos en ubicaciones específicas.
  • PATCH: Aplica parcialmente modificaciones a un recurso existente. Se utiliza para realizar actualizaciones parciales en lugar de reemplazar completamente un recurso.
  • DELETE: Elimina un recurso específico en el servidor.
