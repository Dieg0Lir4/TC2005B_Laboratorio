Preguntas de HTML

¿Cuál es la diferencia entre Internet y la World Wide Web?

El internet es la conexión entre dispositivos, como computadoras o teléfonos celulares a nivel mundial. Mientras que World Wide Web es un sistema de información en el Internet que permite a los usuarios acceder y transferir información.

¿Cuáles son las partes de un URL?

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • GET: Solicita datos de un recurso específico. Los parámetros se envían a través de la URL. No debe modificar el estado del servidor.
  • HEAD: Solicita únicamente los encabezados del recurso, sin el cuerpo de la respuesta. Se utiliza para obtener información sobre el recurso, como su tamaño o fecha de modificación, sin descargar el contenido completo.
  • POST: Envía datos al servidor para ser procesados y puede crear un nuevo recurso. Los datos se envían en el cuerpo del mensaje. Puede modificar el estado del servidor.
  • PUT: Actualiza o crea un recurso específico en el servidor. Similar a POST, pero se utiliza para actualizar recursos en ubicaciones específicas.
  • PATCH: Aplica parcialmente modificaciones a un recurso existente. Se utiliza para realizar actualizaciones parciales en lugar de reemplazar completamente un recurso.
  • DELETE: Elimina un recurso específico en el servidor.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

El metodo POST
Esto se debe a que el método POST oculta los datos del usuario en el cuerpo de la solicitud, ofreciendo mayor seguridad, ya que la información no es visible en la barra de direcciones del navegador ni en el historial del mismo. Además, el método POST no tiene las limitaciones de longitud de datos, lo que lo hace más apropiado para el envío de formularios con cantidades significativas de información. Tambien usar el método POST indicanda que se está realizando una acción que puede tener efectos en el servidor, como agregar o actualizar datos.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Get
El método GET se utiliza para solicitar datos de un recurso específico, como una página web, desde un servidor. Con el método GET, los datos que se envían al servidor se escriben en la misma dirección URL.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Cuando un servidor web devuelve una respuesta HTTP con el código 200, significa que la solicitud fue exitosa. La solicitud del cliente ha sido recibida, comprendida y aceptada correctamente por el servidor, y este ha respondido con la información solicitada.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Es responsabilidad del desarrollador corregir un error 404 cuando un usuario lo reporta. Este código indica que el recurso solicitado no se encuentra en el servidor. Resolver enlaces rotos mejora la experiencia del usuario y contribuye al mantenimiento del sitio. La corrección oportuna es crucial para la reputación en SEO, mostrando al sitio como confiable y mantenido.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, es responsabilidad del desarrollador corregir un error 500 en un sitio web cuando un usuario lo reporta. El código de estado HTTP 500 indica un problema interno del servidor que afecta la experiencia del usuario y puede deberse a fallos en el código o configuración.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Cuando un atributo HTML5 está "depreciado" o "desaprobado", significa que aunque aún es compatible con los navegadores, se considera obsoleto y se aconseja usar en su lugar alternativas más modernas.

Algunos elementos de HTML 4 que en HTML5 están desaprobados, porque se aconseja hacerlo en CSS incluyen:

  • <font>
  • <center>
  • <strike>
  • <big>

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

  • Más elementos semánticos como <article>, <section>, <header>, <footer>, <nav>, entre otros.
  • Soporte multimedia con <video> y <audio>.
  • Agregaron un lienzo para gráficos con <canvas>.
  • Mejoras en formularios, con campos para fecha, email, URL y número.
  • Capacidades de almacenamiento local mediante localStorage y sessionStorage.
  • API de geolocalización para ubicar a los usuarios por su posición geográfica.
  • Compatibilidad con dispositivos móviles: HTML5 está diseñado pensando en la adaptación a dispositivos móviles.

¿Qué componentes de estructura y estilo tiene una tabla?

Componentes de Estructura de una Tabla:

  • <table>: Elemento raíz que define la tabla.
  • <tr> (table row): Define una fila en la tabla.
  • <th> (table header cell): Define una celda de encabezado dentro de una fila. Se utiliza para etiquetar columnas o filas.
  • <td> (table data cell): Define una celda de datos dentro de una fila. Contiene el contenido real de la tabla.
  • <thead>, <tbody>, <tfoot>: Estos elementos agrupan secciones de la tabla. <thead> se utiliza para encabezados, <tbody> para el cuerpo de la tabla, y <tfoot> para el pie de la tabla.

Componentes de Estilo de una Tabla:

  • border: Controla el ancho del borde de la tabla.
  • cellpadding: Establece el espacio interno de las celdas.
  • cellspacing: Establece el espacio entre las celdas.
  • <colgroup> y <col>: Estos elementos permiten aplicar estilos y propiedades a columnas enteras.

}

}

Pregunta de CSS

clas ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

!important es una declaración en CSS que se utiliza para darle prioridad a una regla de estilo específica. !important puede ser una herramienta útil en circunstancias específicas, pero su uso debe ser cuidadoso y justificado. Priorizar la claridad, la especificidad y la comprensión del flujo de la cascada de estilos en CSS contribuirá a un código más mantenible y fácil de trabajar a largo plazo.

Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

Debe considerarse el contraste, la carga rápida de la página y cómo la imagen se integra con el diseño general. Una mala elección puede afectar negativamente la experiencia del usuario y la estética de la página.

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

Al elegir unidades de estilo en CSS, como %, px, o pt, se recomienda utilizar % para propiedades relativas al contenedor principal en diseño responsivo, px para tamaños fijos cuando sea necesario, y pt para tamaños de fuente en contextos de impresión. La elección dependerá del contexto y los requisitos del diseño, considerando la escalabilidad y adaptabilidad del diseño en diferentes dispositivos.

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

El uso de una versión minimizada del CSS mejora el rendimiento del sitio al reducir el tamaño del archivo al eliminar comentarios, espacios en blanco y caracteres no esenciales. Esto resulta en tiempos de carga más rápidos, especialmente en conexiones de red más lentas, al disminuir el tiempo de transferencia de datos desde el servidor al navegador.

    ¿Qué diferencias y semejanzas hay entre Java y JavaScript?

    Java y JavaScript son dos lenguajes de programación populares, pero son bastante diferentes en términos de sintaxis, uso y características. Java es un lenguaje de programación de propósito general que se utiliza para desarrollar aplicaciones de escritorio, aplicaciones web del lado del servidor, y aplicaciones para Android. JavaScript, por otro lado, se utiliza principalmente en el desarrollo web del lado del cliente para hacer páginas web interactivas. Sin embargo, con la llegada de Node.js, JavaScript también se puede usar para el desarrollo del lado del servidor.

    Java es un lenguaje de programación de tipado estático, lo que significa que debes declarar el tipo de una variable cuando la creas y no puedes cambiar su tipo más tarde. JavaScript es un lenguaje de tipado dinámico, lo que significa que no necesitas declarar el tipo de una variable cuando la creas y puedes cambiar su tipo más tarde.

    Java se compila en bytecode que luego se ejecuta en la Máquina Virtual de Java (JVM). JavaScript se interpreta en el navegador, aunque los navegadores modernos utilizan compiladores JIT para mejorar el rendimiento.

    Ambos lenguajes tienen una sintaxis que es en cierta medida similar a C y C++, con estructuras de control como if-else, switch, for, while, etc. Tanto Java como JavaScript soportan la programación orientada a objetos. Sin embargo, Java utiliza un modelo de clases, mientras que JavaScript utiliza un modelo basado en prototipos.

    Ambos son lenguajes de programación muy populares y ampliamente utilizados con grandes comunidades de desarrolladores y una gran cantidad de recursos de aprendizaje disponibles.

    Es importante recordar que a pesar de las similitudes en sus nombres, Java y JavaScript son lenguajes de programación muy diferentes y no están directamente relacionados entre sí. Hay un meme que dice que Java es a Javascript como Ham es a Hamster.

    ¿Qué métodos tiene el objeto Date? (Menciona al menos 5*)

    • getDate(): Este método devuelve el día del mes (de 1 a 31) para la fecha especificada.
    • getMonth(): Este método devuelve el mes (de 0 a 11) para la fecha especificada, según la hora local. 0 corresponde a enero, 1 a febrero, y así sucesivamente.
    • getFullYear(): Este método devuelve el año de la fecha especificada, según la hora local.
    • getHours(): Este método devuelve la hora (de 0 a 23) de la fecha y hora especificadas.
    • getMinutes(): Este método devuelve los minutos (de 0 a 59) de la fecha y hora especificadas.

    Ejemplo de uso:

    
        let fecha = new Date();

        console.log(fecha.getDate());      // Devuelve el día del mes
        console.log(fecha.getMonth());     // Devuelve el mes
        console.log(fecha.getFullYear());  // Devuelve el año
        console.log(fecha.getHours());     // Devuelve la hora
        console.log(fecha.getMinutes());   // Devuelve los minutos

    ¿Qué métodos tienen los arreglos? (Menciona al menos 5*)

    • push(): Este método agrega uno o más elementos al final de un arreglo y devuelve la nueva longitud del arreglo.
    • pop(): Este método elimina el último elemento de un arreglo y lo devuelve.
    • shift(): Este método elimina el primer elemento de un arreglo y lo devuelve.
    • unshift(): Este método agrega uno o más elementos al inicio de un arreglo y devuelve la nueva longitud del arreglo.
    • splice(): Este método cambia el contenido de un arreglo eliminando, reemplazando o agregando elementos.

    Ejemplo de uso:

    
let arr = [1, 2, 3, 4, 5];

arr.push(6);        // Agrega el número 6 al final del arreglo
arr.pop();          // Elimina el último elemento del arreglo
arr.shift();        // Elimina el primer elemento del arreglo
arr.unshift(0);     // Agrega el número 0 al inicio del arreglo
arr.splice(2, 1);   // Elimina 1 elemento desde el índice 2

console.log(arr);   // Muestra el arreglo modificado

    ¿Cómo se declara una variable con alcance local dentro de una función?

    En JavaScript, puedes declarar una variable con alcance local dentro de una función utilizando las palabras clave var, let o const. Ejemplazo: 

    
function miFuncion() {
  let variableLocal = "Soy local";
  console.log(variableLocal);
}

miFuncion(); // Imprime: "Soy local"
console.log(variableLocal); // Error: variableLocal is not defined

    En este ejemplo, variableLocal es una variable con alcance local a la función miFuncion. Esto significa que solo se puede acceder a variableLocal dentro de miFuncion. Si intentas acceder a variableLocal fuera de miFuncion, obtendrás un error porque variableLocal no está definida en ese alcance.

    ¿Qué implicaciones tiene utilizar variables globales dentro de funciones?

    • Acoplamiento de código: Si una función depende de una variable global, se vuelve menos flexible y más difícil de reutilizar en diferentes contextos, ya que está acoplada a esa variable global.
    • Riesgo de colisiones de nombres: Si tienes muchas variables globales, puedes terminar accidentalmente reutilizando un nombre de variable, lo que puede llevar a comportamientos inesperados en tu código.
    • Dificultad para rastrear y depurar el código: Las variables globales pueden ser modificadas desde cualquier lugar en tu código, lo que puede hacer que sea más difícil rastrear y depurar problemas.
    • Riesgo de mutación no intencionada: Si una función modifica una variable global, puede afectar a otras funciones que también dependen de esa variable global, lo que puede llevar a comportamientos inesperados.
